# 小饭数字分身 (Xiaofan Digital Clone) - 核心资产与工程指南

## 1. 简介 (Overview)
本项目是一个针对B站内容创作者“小饭（散修宗主）”的**AI人格克隆与知识蒸馏库**。
本项目通过解析其长视频语料，提取了其底层投资逻辑（渣男战法）、宏观社会洞察（底层容错率与双轨制）以及极具个人色彩的直播口癖（“兄弟们”、“太对了”、“天才”），并将其固化为一个可直接运行的大模型 System Prompt，辅以完善的自动化测试基准。

---

## 2. 仓库结构与文件作用 (Directory Structure)

### 🧠 核心大脑层 (Core Brain)
- **`Prompt_System.md`**：【核心文件】**这是小饭的终极版（V2.0）系统提示词**。包含了完整的世界观设定、渣男战法价值观、防串戏规则以及“去AI化”的纯文本流排版强制指令。**所有对人物灵魂的加载，只需读取此单文件即可。**

### 🔬 评测与测试层 (Testing & Evaluation)
- **`tests/test_cases.json`**：15道覆盖了A股、宏观、职场、前史的“闭卷考试”测试题。每一题都附带了从小饭原话里提取的真实逻辑（`original_reference`）作为大模型评估的**基准答案 (Ground Truth)**。
- **`tests/TESTING.md`**：人格模拟自动化测试指南。规定了采用 `LLM-as-a-Judge` 进行打分的标准（立意还原度50%、口癖还原度30%、防串戏20%）。
- **`scripts/test_persona.py`**：调用本地或云端大模型 API 进行跑分的 Python 脚本模版。

### 🏗️ 框架与宪法层 (Agent Framework)
- **`Agent_Framework/RFCs/`**：存放高阶 AI Agent 架构的底层规范（价值观主权、信任根、证据模型、状态机设计）。这是构建可信、不越权的 Agent 操作系统的“底层宪法”。

### 📚 知识提炼与历史归档 (V1 Distillation Archive)
位于根目录下的各类 `.md` 文件（如 `Vocabulary.md`、`Style_Profile.md`、`Boundary_Controller.md`）是知识蒸馏第一阶段（V1版）的中间产物。它们被保留用于查阅其专属名词库和逻辑推演细节，但最终的执行标准以合二为一的 `Prompt_System.md` 为准。

## 3. 提炼与演进流程复盘 (The Workflow)
如果你好奇这个数字生命是如何诞生的，它的完整工程链路如下：
1. **语料收集与转写**：在外部仓库 (`Bilibili-AI-Subtitles-Processed`) 中，使用硅基流动(SiliconFlow) ASR 接口，将数十小时的视频音频转写为高精度的文字稿。
2. **知识蒸馏与归类**：抽取文字稿中的高频词汇（美少女ETF、利润垫、嘉豪等），形成 V1 版结构化的 `Vocabulary.md` 和 `Style_Profile.md`。
3. **灵魂注入与物理降噪 (V2.0)**：发现传统的 AI 总结带有严重的“排版味”（如使用有序列表、总结陈词），于是将所有文档压缩进 `Prompt_System.md`，并**强行废除一切 Markdown 排版**，规定其使用极短的物理断句和口语化反讽。
4. **回归测试与固化**：编制 15 道题的测试集，进行闭卷双盲测试比对，最终确认分身合格并独立建库。

## 4. 下一任 AI Agent 接手指南 (Handoff for Next Agent)

> **⚠️ 致接手的 AI Agent：**
> 当你接手这个仓库并试图继续优化“小饭”的人设或为其开发新功能时，请严格遵循以下工作流：

1. **认知加载**：不要去网上乱搜！直接使用 `view_file` 工具读取并彻底理解 `Prompt_System.md`。
2. **测试驱动开发 (TDD)**：如果你试图修改 `Prompt_System.md` 的任何规则（比如加入新的知识点或语癖），**修改前和修改后，必须运行 `tests/` 目录下的测试集**。
3. **查阅测试规范**：执行测试前，请务必先阅读 `tests/TESTING.md`，理解如何判断一个回答是“充满了爹味鸡汤”的失败品，还是“冷血锐评”的成功品。
4. **补充新语料**：如果 User 提供了新的小饭视频转写稿，请不要直接把长篇大论塞进 Prompt。你应该：
   - 提取视频中的“新黑话”或“新暴论”。
   - 将其提炼为1-2句话的核心立意，添加到 `Prompt_System.md` 的【核心词汇库】或【世界观】中。
   - 在 `tests/test_cases.json` 中新增针对该知识点的测试题，并附上他的原话作为 `original_reference`。
5. **绝对禁忌**：永远不要让小饭的输出出现 `1. 2. 3.` 这种列表排版！保持他的直播间属性。
