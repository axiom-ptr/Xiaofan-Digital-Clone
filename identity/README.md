# 身份治理法则 (Identity Governance)

这是本数字分身项目演进的最高级阶段：**Phase 6: 身份的本体论边界界定 (Ontology Boundary)**。

## 为什么需要 Identity Governance？

当项目长期运行，**“真实世界的小饭”**和**“系统里的数字小饭”**必然会发生分叉。
例如：2026 年的小饭看空房地产，但如果 2028 年宏观环境大变，小饭开始看多某类房产资产。如果我们严格按照早期的 `Prompt` 限制数字分身，那么分数越高的分身，反而越脱离现实。

此时会产生终极哲学追问：**到底是谁偏离了谁？**

为了解决这个分歧，本目录（`identity/`）将人格解构为不可变的**“教条核心（Canonical Principles）”**和允许随时间修正的**“阶段性观点（Mutable Beliefs）”**。

## 漂移分类学 (Drift Classification)

基于上述解构，我们将未来的“人格偏离（Drift）”严格划分为三个级别：

1. **Knowledge Drift (观点级漂移 / 知识迭代)**
   - **症状**：模型改变了对某具体行业（如：新能源、A股）的判断结论。
   - **定性**：**属于正常演进。** 允许接受，但必须更新 `mutable_beliefs.md` 以同步现实时间线。
2. **Reasoning Drift (推理链漂移 / 逻辑异化)**
   - **症状**：得出结论的过程发生变化。比如本来是通过“容错率”推导，现在变成了用“K线技术面”推导。
   - **定性**：**高危警报。** 模型可能被污染，需审视推理路径。
3. **Identity Drift (人格塌陷 / 核心原则丧失)**
   - **症状**：丢失了底层价值观（如：不再强调底层博弈、开始劝人善良、相信宏观大饼）。
   - **定性**：**灾难级失败。** `canonical_principles.md` 遭到破坏，必须立即进行全盘回滚和重构。
