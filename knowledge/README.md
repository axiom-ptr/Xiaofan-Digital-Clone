# 知识时效层 (Knowledge Layer)

> **核心原则**：人格层（`persona/`）永久有效；知识层（`knowledge/`）可随时间更新，不影响人格本身。

---

## 为什么要分离知识层？

投资主播有一个天然问题：**人格 ≠ 知识**。

小饭在2024年对房地产的判断，到2028年可能已经过时。但他的"渣男战法"世界观、他看市场的底层逻辑、他的表达风格——这些是永久的人格资产，不随时间过时。

如果把时效性观点直接写进 `persona/`，会导致数字分身越来越像"历史标本"。

---

## 目录结构说明

```
knowledge/
├── README.md                    # 本文件，维护规范
├── macro_2024.md                # 2024年宏观观点快照
├── macro_2025.md                # 2025年宏观观点快照（待补充）
└── stock_views/
    └── etf_strategy_2024.md     # ETF操作策略（可随行情更新）
```

---

## 如何使用知识层

1. **加载时**：将 `persona/` 作为 System Prompt 的**主体**，`knowledge/` 中的相关文件作为**补充上下文**附加到 System Prompt 末尾或 User Prompt 中
2. **更新时**：直接新增或替换 `knowledge/` 下的快照文件，不需要动 `persona/` 任何内容
3. **归档时**：旧版快照不删除，重命名加年份后缀保留（如 `macro_2023_archived.md`）

---

## 知识层 vs 人格层 对比

| 属性 | 人格层 (`persona/`) | 知识层 (`knowledge/`) |
|---|---|---|
| 生命周期 | 永久 | 随时间更新 |
| 修改频率 | 极少（核心变更需跑回归测试） | 可频繁更新 |
| 修改后需回归测试 | 是 | 否（但建议验证） |
| 内容类型 | 价值观、口癖、世界观、禁忌 | 市场观点、时效数据、行情判断 |
